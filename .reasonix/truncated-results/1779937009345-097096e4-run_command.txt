## 卷1：夜班便利店

### 卷1·第1章｜规则的第一夜

周仓在無街道、無出口的便利店中醒來，讀到七條《夜班便利店守則》，取得臨時工牌 `F-0217`，並在收銀機底座下發現寫著「陈述」的紙條。冰櫃異響觸發「心跳倒數」的被動預警，代價表現為右耳耳鳴與右手顫抖。紅雨衣巡邏員出現在門外後被系統修正為非有效交易對象，真正的第二名顧客抵達。

### 卷1·第2章｜牛奶与红纸

許小小以空牛奶瓶與紅紙片購買牛奶。周仓依規則五拒絕紅紙，避免觸碰污染掌印；彈幕系統首次以文字模式介入，提醒不要碰。許小小改用 3 分積分幣完成牛奶購買，周仓獲得 3 積分並解鎖道具商城預覽。紅雨衣人再次靠近，系統顯示工牌暴露度，周仓確認紅雨衣人的識別重點可能是工牌反光。

### 卷1·第3章｜红雨衣人巡逻

孕婦進店要求購買黑傘 `E-0217`，提出以死胎紀念幣與影子作為代價。交易進入待審核狀態。紅雨衣人的巡邏與工牌識別持續逼近，周仓從冰櫃中取得陳述的工牌，確認 `F-0217` 更像職位 / 坑位編號，而不是個人身份。周仓利用陳述工牌轉移紅雨衣人的注意力。冰櫃黑水開始朝固定方向流動，規則六被改寫，新增「不要直視超過三秒」。

### 卷1·第4章｜黑伞

凌晨三點後門自動打開，孕婦重新要求黑傘。孕婦詢問周仓名字時觸發心跳倒數，證明「名字」是危險點。孕婦的影子能物理接觸環境，甚至威脅收銀台電源開關。店長從門口進入，帶有管理權限，彈幕暫停，黑傘停止震動；店長的出現強制中斷黑傘交易，顯示他不希望這筆交易以當前方式完成。

### 卷1·第5章｜灯光下

孕婦告訴周仓，照片中男人不是店長，而是上一任與 `F-0217` 有關的人。周仓檢查工牌，發現工牌夾內側刻有「不要相信店长」。工牌夾位置與凹痕偏移，周仓推定有人動過工牌，嫌疑指向店長；姓名欄短暫出現可編輯狀態，暗示主播身份可能被修改。便利店燈光範圍縮小，黑水避開光照並等待光線退到指定位置，確認燈光具有庇護 / 界線功能。

### 卷1·第6章｜未登记身份

周仓發現黑傘交易的買方欄位是「未登記身份」，並將買方從孕婦改成「未登記身份（附屬物）」，也就是孕婦的影子。系統接受影子作為買方，交易條件變成死胎紀念幣與買方自身抵押，並保留栖息權倒數。孕婦離開便利店，影子被留在店內作為真正買方。黑水從燈光邊界向影子方向移動。

### 卷1·第7章｜打烊倒数

周仓把黑傘移到影子接觸範圍內，影子轉移到黑傘上，特殊交易完成；黑傘變成灰傘並被放回伞架。店長以玻璃反光 / 門內陰影的方式出現在正門旁，無聲說出「不是现在」，顯示他仍監看交易結果。系統判定黑傘交易為特殊交易，不計入新人基礎考核，考核仍只有 1/3。倒數剩下數分鐘時，許小小提示周仓完成退貨與空瓶回收。

### 卷1·第8章｜结算

周仓打開牛奶盒，發現盒內沒有牛奶，而是貼著先前付款用的 3 分積分幣；許小小提示「買了，要退。瓶子，也要還」。周仓完成異常空牛奶盒退貨，取回 3 分幣，再回收玻璃空牛奶瓶，取回 1 分押金，使新人考核達成 3/3。副本 `C-0217` 結算完成，評價 B-，積分獎勵 15，當前積分 22，並解鎖三項道具。周仓離開便利店，進入休息艙 / 過渡空間。屏幕顯示陳述留言：「你拿到了编号三，做得不错。但现在不要打开那张照片。」直播圖標仍亮，倒數顯示約三天後進入下一次副本。

## 跨副本可延續的規則型概念

### 1. 名字 / 身份危險

**來源：** 卷1·Ch4+Ch5  
**可延續：** 是  
**用法：** 第二卷可以把名字、票根、座位號、字幕身份欄、本人確認等設計成危險點。  
**限制：** 不可直接說「知道真名就會死」，除非新副本規則正式建立。

### 2. 假規則 / 隔離規則

**來源：** 卷1·Ch1+Ch2，「請勿讓她確認你看得見她」  
**可延續：** 是  
**用法：** 第二卷可以揭示某些系統提示不一定是保護主播，而是為了隔離潛在隊友、阻止關係建立、讓主播只把對方當作「交易對象」或「未確認身份」。  
**限制：** 不可直接說所有規則都是假的。假規則必須有可回推證據，例如：遵守後讓周仓失去盟友資訊；違反後沒有懲罰；許小小能用行動證明她不是敵人；彈幕或陳述留下反向提示。

### 3. 可見性 / 身份確認危險變體

**來源：** 卷1·Ch1+Ch2 的表層提示  
**可延續：** 可以作陷阱或變體  
**用法：** 第二卷仍可設計「不要確認銀幕中的自己」「不要讓觀眾席確認你聽得見」等規則，但必須讓讀者不確定它是真規則還是假規則。  
**限制：** 不可再把「周仓不能承認看得見許小小」當成硬限制。許小小預定為卷2隊友，周仓可以逐步直接與她互動，但最好在正文中先讓他意識到第一副本提示可能是誘導。

### 4. 規則文字不可靠

**來源：** 規則六污染、結算記錄「規則污染 ×1」、假規則候選  
**可延續：** 是  
**用法：** 放映廳字幕、票根、銀幕規則可被污染、替換，或故意用半真半假的方式誘導。  
**限制：** 不可讓所有規則都任意改變；每次改變需有可觀察痕跡或後續代價。

### 5. 交易不只買賣

**來源：** 牛奶銷售 / 退貨 / 回收、黑傘特殊交易  
**可延續：** 是  
**用法：** 第二卷可以把回答問題、提交片名、確認身份寫成「交易 / 補錄 / 歸檔」。  
**限制：** 必須有系統欄位或結算提示支撐。

### 6. 觀察比解釋重要

**來源：** 周仓以燈光、黑水、工牌、買方欄位推理  
**可延續：** 是  
**用法：** 第二卷應讓周仓透過字幕延遲、觀眾席反應、票根細節推理。  
**限制：** 不可讓 AI 直接旁白解釋真相。

=== 小說品質規準 ===
# Story Quality Rubric｜小說品質評分規準

> 目的：讓系統不只檢查「有沒有 bug」，也檢查「這章是否好看、是否推動長篇」。  
> 使用方式：AI 寫章前先讀本檔；寫完後用 `_腳本/evaluate_story_quality.py` 產生品質報告，再決定是否重寫。

---

## 10 個核心要素

| 編號 | 要素 | 判斷問題 | 低分表現 | 高分表現 |
|------|------|----------|----------|----------|
| Q1 | 一句話鉤子 | 本章是否有一個讓讀者想知道答案的核心問題？ | 只有事件，沒有疑問 | 一開章或章末有明確懸念 |
| Q2 | 明確危機 | 是否有外部危機、內在危機或世界危機？ | 只是聊天 / 資料交代 | 危機具體、正在逼近、有時間壓力 |
| Q3 | 有矛盾的主角 | 主角是否有傷口、慾望、恐懼或道德矛盾？ | 主角只是冷靜解題 | 主角的選擇暴露內在缺口 |
| Q4 | 清楚短期目標 | 本章 / 本段現在要解決什麼？ | 行動目的模糊 | 讀者知道「現在必須做什麼」 |
| Q5 | 長期主線謎團 | 是否推進身世、世界真相、終極問題？ | 和主線無關 | 至少新增 / 推進一個長線疑問 |
| Q6 | 世界觀規則可運作 | 規則、能力、限制是否真的影響劇情？ | 規則只是背景 | 主角用規則行動，規則產生代價 |
| Q7 | 配角有功能與立場 | 配角是否有自己的功能、立場、秘密？ | 配角只捧主角 | 配角改變局面，且不完全可控 |
| Q8 | 高潮有選擇與代價 | 高潮是否是選擇，不只是打贏？ | 主角輕鬆過關 | 主角必須取捨，勝利留下後果 |
| Q9 | 真相分層揭露 | 揭露是否改變讀者理解？ | 一次性說明設定 | 每次只揭一層，並重排前文意義 |
| Q10 | 主題深度 | 本章是否觸及情感 / 問題 / 人性？ | 只有設定好玩 | 讀者記住的不只是機制，還有問題 |

---

## 評分標準

| 分數 | 定義 |
|------|------|
| 0 | 完全沒有 |
| 1 | 有一點痕跡，但不清楚 |
| 2 | 有，且可被讀者感知 |
| 3 | 很強，是本章主要亮點 |

總分滿分 30。

| 總分 | 判定 |
|------|------|
| 0–12 | 需要大修 |
| 13–18 | 可讀，但偏平 |
| 19–24 | 合格，有追讀力 |
| 25–30 | 強章，建議保留骨架 |

---

## 類型加權：規則怪談 / 無限流直播

本作品類型不是普通都市文，評分時額外重視：

1. **規則是否可操作**：讀者要能看到主角如何利用規則。
2. **系統是否冷酷但有邏輯**：系統不能硬拗，判定必須有可回推依據。
3. **爽點是否有代價**：主角贏一手後，要留下陰影或後果。
4. **伏筆是否能回頭理解**：讀者回看前文時，應覺得「原來如此」。
5. **配角不是工具人**：許小小、店長、陳述、孕婦都應有自己的不可知立場。

---

## 寫作前必問

AI 生成章節前，先回答：

- 本章一句話鉤子是什麼？
- 本章短期目標是什麼？
- 本章主角要做哪個選擇？代價是什麼？
- 本章新增或回收哪個伏筆？
- 本章配角的功能與立場是什麼？
- 本章最後一句是否能推動下一章？

---

## 寫完後必查

AI 生成章節後，檢查：

- 是否有至少一個明確危機？
- 是否有一個規則被使用、驗證、污染或改寫？
- 是否有一個爽點或反轉？
- 是否有一個代價或殘留後果？
- 是否有一句可以作為章末鉤子的收束？
- 是否只是作者解釋，而不是角色觀察與系統結果？

---

## 重寫原則

若章節分數低，不要整章推倒，優先做局部修補：

1. 加強章首或章末鉤子。
2. 把作者解釋改成角色觀察。
3. 把「得到物件」改成「完成交易 / 付出代價 / 改變狀態」。
4. 讓配角用一句話或一個動作改變局面。
5. 讓系統只給結果，不替作者解釋。
6. 把爽點後面補一個短暫陰影。

=== 故事聖經 ===
# 故事圣经

## Logline（一句话）
一个在直播间副本中靠规则生存的便利店店员，发现真正可怕的不是怪物——而是自己正在丧失对"正常"的判断力。

## 核心主题
- 理性 vs 疯狂的界线：遵守规则能活，但规则本身可能是陷阱
- 人性 vs 异化：使用能力 = 让自己更接近怪物
- 交易的代价：每一次交易都在付出某种东西

## 目标读者情绪
- 第一章让读者感到：不安、好奇、规则的压迫感
- 中段让读者感到：紧张、对主角"到底是不是人类"的怀疑
- 结局让读者感到：震撼、回味、对规则的全新理解

## 不可打破的规则（Story Rules）
1. 主角不能靠运气获胜，必须靠规则推导和观察细节
2. 每个副本的规则必须有陷阱（省略、矛盾、过时、钓鱼）
3. 每章结尾必须有钩子（悬念/反转/升级/预示/角色危机）
4. 能力的代价必须随着使用次数累积
5. 恐怖来自"日常中的异常"，而非 Jump scare

## 氛围基调
- 恐怖类型：规则怪谈 + 精神污染
- 参考风格：从红月开始（黑山老鬼）
- 叙事节奏：宁静期60% → 信号期25% → 爆发期15%
  - 注意：从红月的核心是"60%篇幅建立正常假象"，不要缩短宁静期
  - 异常信号出现后不解释，让读者自行脑补

## 写作风格参考（从红月风格）

详细手法分析见以下两个文件（按优先级排列）：
1. `_共用知识库/写作手法/从红月写作技法大全.md`（综合分析，基于137章免费章节）
2. `_共用知识库/写作手法/从红月手法.md`（早期拆解）

### 核心叙事模型：正常中的异常

从红月最核心的策略：**恐怖不是突然闯入，而是从日常中自然浮现。**

```
阶段1 日常层（60%）：完全正常的场景，让读者放下戒备
阶段2 信号层（25%）：插入微小异常，不解释，继续叙事
阶段3 爆发层（15%）：异常累积到临界点后释放
```

**关键：不解释异常。** 妈妈为什么带剪刀出门？陆辛不问。血迹从哪来？陆辛不问。读者自己去想。

### 主语控制（最重要！）

从红月解决了第三人称通病的核心技法：

```
/ 连续不超过2句以"他"开头 / 平均每4-5句只出现1次"他"

/ 替代方案（按优先级）：
1. 环境/感官开句：暗红色的月亮低垂在城市上空……
2. 动作链串联：他走过了台阶，穿过了站台，到了地面，抬头看去……
3. 对话打断：他走向吧台。"有人让我把这份文件送过来。"
4. 配角独立出场：妹妹正卧在沙发上，抱着零食……
5. 物品/细节开句：桌上已经摆了四副碗筷……
```

### 角色塑造：异常冷静的观察者

- 主角从不展示恐惧，对恐怖场景的反应是"麻烦"和"尴尬"而非"恐惧"
- 每个角色都有两面（表面版本 vs 真实版本），割裂感制造恐怖
- 配角出场时不经过主角中介（不写"他看见妹妹在…"，直接写"妹妹正卧在沙发上…"）

### 叙事技法

- **展示不解释**：能力/规则通过应用场景自然呈现，不用文字说明
- **细节代替心理描写**：展示一个细节（血迹/气味/温度），不加评语，让读者推理
- **视角切换制造信息差**：读者比主角知道得多（关键节点切到上帝视角，每次不超过200字）
- **对话三重功能**：每句对话同时推进情节 + 塑造角色 + 暗示隐藏信息

### 精神污染写作法
- 每一章必须同时有外层冲突（剧情推进）+ 内层冲突（心理压力）
- 使用能力必有代价（生理+心理+累积效应）
- 主角对恐怖场景的反应"异常平静"，暗示他本身有问题

### 规则型恐怖手法
- 规则三层次：表层（明文）→ 暗层（推导）→ 核心（底层逻辑）
