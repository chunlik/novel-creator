# 章節生成 Prompt

> 用途：根據 `compile_context.py` 輸出的完整上下文，生成新章節初稿。  
> 適用：新章節第一次生成。

---

## 使用方式

先執行：

```bash
python _腳本/compile_context.py --novel {novel_id} --chapter <章號> --vector-query "<本章核心關鍵詞>"
```

把輸出的完整上下文貼給 AI，然後追加以下 prompt。

---

## Prompt

```text
請根據以上完整上下文，生成本章完整 Markdown 章節。

硬性要求：
1. 嚴格遵守全局摘要、時間線、系統狀態、角色狀態、物品連續性、規則連續性、場景機制、伏筆管理。
2. 不可使用未落地的 `pending_canon` 設定；如果本章需要使用，必須在正文中先正式落地。
3. 不可改變當前積分、物品持有者、角色狀態、已歸檔副本狀態，除非本章正文清楚寫出事件原因。
4. 正文使用簡體中文與大陸式雙引號 `""`。
5. 系統只給結果、規則、提示或錯誤訊息，不替作者解釋世界觀。

小說品質要求：
1. 本章必須有一句話鉤子，讓讀者想知道答案。
2. 本章必須有明確短期目標。
3. 本章至少推進一個長期主線謎團。
4. 本章至少讓一條規則 / 機制被使用、驗證、污染或改寫。
5. 本章高潮必須包含選擇與代價，不要只靠主角輕鬆成功。
6. 如果有爽點，爽點後面要留下陰影或後果。
7. 配角不能只是工具人；配角要有自己的功能、立場或不可知性。
8. 章末必須留下具體鉤子，而不是單純停在氣氛上。

輸出格式：
- 保留 frontmatter。
- 正文不要出現作者解釋，例如「第二章」「伏筆」「讀者會知道」。
- 用角色觀察、行動、對話、系統提示來呈現資訊。
- 輸出完整章節，不要只輸出大綱。
```
