# 章节检查清单

每章写完后，逐项确认。优先级从高到低排列。

## 一、主语控制（从红月核心技法，最难）
- [ ] 连续不超过 2 句以"他"开头
- [ ] 通篇平均每 4-5 句只出现 1 次"他"
- [ ] 每段至少 1 句不以人称开句（环境/感官/物品/对话开头）
- [ ] 无"他发现/他看到/他听到/他知道"类中介语（直接写事实）
- [ ] 动作链串联：用逗号连接 3-4 个动作，不重复主语

## 二、正常中的异常（从红月核心叙事模型）
- [ ] 开场 60% 篇幅用于建立"正常"假象
- [ ] 至少 3 个微型异常信号累积后才爆发
- [ ] 异常信号出现后不解释，让读者自行推理
- [ ] 恐怖来自日常细节中的异常（而非直接描写怪物）

## 三、双层冲突
- [ ] 外层冲突（剧情推进/规则破解）
- [ ] 内层冲突（精神腐蚀/能力代价）
- [ ] 主角对恐怖场景的反应异常冷静（非恐惧，而是"麻烦"/"困扰"）

## 四、叙述技法
- [ ] 配角出场不经过主角中介（直接写配角动作/对话）
- [ ] 对话至少承担 2 重功能（推情节+塑角色+暗示信息）
- [ ] 细节代替心理描写（展示事实，不做评论）
- [ ] 角色有"表面版本 vs 真实版本"的割裂感

## 五、氛围检查
- [ ] 感官聚焦：同一时刻不超过 2 种感官
- [ ] 不可名状展示：怪物最多 3 个感官细节，其余留白
- [ ] 至少一个让读者"心里发毛"的细节

## 六、逻辑验证（新增，每章必查）
- [ ] 数字准确性：字数、时间、价格、倒计时数字是否正确
- [ ] 因果链：每一步动作都有前因后果，无跳跃
- [ ] 指代明确：每个动作/行为必须有所属主语，无悬浮动作（如"一只手搭上肩"——谁的手？为什么？）
- [ ] 空间一致性：人物位置、物品状态、视角切换符合物理逻辑
- [ ] 技术细节：收银机/系统/工具的行为模式前后一致

## 七、规则检查
- [ ] 规则被验证或被打破
- [ ] 有新规则被发现
- [ ] 规则之间无矛盾

## 八、章末钩子
- [ ] 中断型/反转型/升级型/预示型/角色型（至少 1 种）

## 九、角色检查
- [ ] 角色反应符合设定
- [ ] 有配角的"人性闪光时刻"
- [ ] 怪物行为有自洽逻辑
