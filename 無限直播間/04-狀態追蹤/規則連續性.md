## 跨副本可延續的規則型概念

### 1. 名字 / 身份危險

**來源：** 卷1·Ch4+Ch5  
**可延續：** 是  
**用法：** 第二卷可以把名字、票根、座位號、字幕身份欄、本人確認等設計成危險點。  
**限制：** 不可直接說「知道真名就會死」，除非新副本規則正式建立。

### 2. 可見性 / 身份確認危險

**來源：** 卷1·Ch1+Ch2，「請勿讓她確認你看得見她」  
**可延續：** 是，但只能作變體  
**用法：** 第二卷可把「不要確認銀幕中的自己」「不要讓觀眾席確認你聽得見」「不要與未登記身份對視」設計成新副本規則。  
**限制：** 不可把許小小規則原封不動搬到放映廳；必須由票根、字幕、座位提示或系統公告重新建立。涉及許小小本人時，周仓仍應避免直接承認看得見 / 聽得見她，除非正文先解除或改寫此限制。

### 3. 規則文字不可靠

**來源：** 規則六污染、結算記錄「規則污染 ×1」  
**可延續：** 是  
**用法：** 放映廳字幕、票根、銀幕規則可被污染或被替換。  
**限制：** 不可讓所有規則都任意改變；每次改變需有可觀察痕跡。

### 4. 交易不只買賣

**來源：** 牛奶銷售 / 退貨 / 回收、黑傘特殊交易  
**可延續：** 是  
**用法：** 第二卷可以把回答問題、提交片名、確認身份寫成「交易 / 補錄 / 歸檔」。  
**限制：** 必須有系統欄位或結算提示支撐。

### 5. 觀察比解釋重要

**來源：** 周仓以燈光、黑水、工牌、買方欄位推理  
**可延續：** 是  
**用法：** 第二卷應讓周仓透過字幕延遲、觀眾席反應、票根細節推理。  
**限制：** 不可讓 AI 直接旁白解釋真相。

## 不可跨副本直接使用的規則

- 不可把「黑水不能進燈光」直接套到記憶放映廳，除非放映廳重新建立光線規則。
- 不可讓紅雨衣人在第二卷繼續按 15 分鐘巡邏，除非卷2重新建立巡邏機制。
- 不可讓關東煮機器、後門、便利店貨架規則直接出現在卷2，除非作為放映畫面或回放記憶。
- 不可讓便利店收銀機交易流程完整複製到放映廳；放映廳應有自己的提交 / 問答 / 歸檔機制。
- 不可讓「請勿讓她確認你看得見她」原句無交代套到所有女性 / 小孩角色；它目前正式適用於許小小與未確認身份顧客，可作跨副本變體但需重新建立。

## 第二卷寫作檢查

寫卷2時，每章至少檢查：

- 本章用到的規則是否已在正文呈現？
- 是否把便利店規則錯套到放映廳？
- 是否讓規則任意改變但沒有可觀察痕跡？
- 是否讓系統解釋太多？
- 是否把 `planning` 規則寫成讀者早已知道？
- 是否把 `pending_canon` 規則寫成已發生？
- 若許小小或未確認身份實體出現，周仓是否直接承認看得見 / 聽得見？若是，正文是否先解除或改寫該限制？
