# AI 導航索引卡

> **使用規則：** 接到任務 → 查下方對應行 → 只讀「必讀」欄 → 視需要讀「參考」欄 → 執行 → 更新「事後更新」欄。
> 不要因為「以防萬一」而多讀檔案。信任這張表。

---

## 一、權威資料來源（精簡 4 層）

寫作時判斷故事事實的採信順序：

| 優先級 | 來源 | 說明 |
|--------|------|------|
| **L1 最高** | `03-章節/` 已寫章節 | 已完成的正文是最高敘事事實 |
| **L2** | `04-狀態追蹤/` 角色狀態機 + 物品連續性 + 伏筆管理 | 跨章約束與當前狀態 |
| **L3** | `02-大綱/` 卷大綱 + 副本設計 | 計畫與方向；若與已寫章節衝突，以章節為準 |
| **L4 參考** | `05-AI上下文/` 當前寫作包 + 下一章指令 + 風格規則 + 禁止事項 | 當前工作包與寫作限制 |

## 四、最少讀取原則

| 任務類型 | 最少讀取檔案數 | 具體 |
|----------|:-----------:|------|
| 寫一章 | **4 必讀 + 0~4 參考** | 卷大綱 + 下一章指令 + 風格規則 + 禁止事項 + 章節範本 |
| 封卷 | **3 必讀** | 本卷全部章節 + 伏筆管理 + 角色狀態機 + 物品連續性 |
| 開新卷 | **3 必讀** | 前卷總結 + 狀態三件套（角色/物品/伏筆） + 副本設計 |
| 修正小錯誤 | **1** | 直接讀目標章節 |
| 回答故事問題 | **1~2** | 全局摘要 + 必要時讀指定章節 |

## 五、寫章後必更新的 3 個狀態檔

> ⚠️ 這三步和跑腳本一樣重要，不可省略。

1. **`04-狀態追蹤/角色狀態機.md`** — 在 `狀態變化記錄` 表格新增一行
2. **`04-狀態追蹤/物品連續性.md`** — 更新物品的 `當前位置` 與 `狀態`
3. **`04-狀態追蹤/伏筆管理.md`** — 新伏筆標 `active`，已回收標 `paid`

## 六、快速自查（寫完章後 30 秒）

- [ ] 主語控制：連續不超過 2 句以「他」開頭？
- [ ] 引號：全部使用 `""`（非 `「」`）？
- [ ] 簡體中文：無繁體字/台灣用詞？
- [ ] 異常不解釋：沒有讓角色或敘述者解釋世界觀？
- [ ] 數字準確：積分、時間、倒數與狀態檔一致？
- [ ] 章末鉤子：有中斷/反轉/升級/預示/角色型鉤子？
