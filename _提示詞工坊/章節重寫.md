# 章節重寫 Prompt

> 用途：根據 `evaluate_story_quality.py` 與 `validate_chapter_logic.py` 的報告，局部重寫已生成章節。  
> 適用：初稿已完成，但需要修 bug、加爽點、補鉤子、改掉作者解釋。

---

## 使用方式

先執行：

```bash
python _腳本/validate_chapter_logic.py --novel infinite_livestream --volume <卷號> --chapter <章號> --write-report
python _腳本/evaluate_story_quality.py --novel infinite_livestream --volume <卷號> --chapter <章號> --write-report
```

把以下內容一起貼給 AI：

1. 原章節全文
2. `06-審校/章節邏輯驗證.md`
3. `06-審校/章節品質評估.md`
4. 以下 prompt

---

## Prompt

```text
請根據邏輯驗證報告與品質評估報告，對本章做「局部重寫」，不要推翻已成立的劇情骨架。

重寫優先級：
1. 先修 ERROR 級邏輯問題。
2. 再修會讓讀者出戲的作者解釋、章節視角、硬拗設定。
3. 再加強低分品質項，例如鉤子、危機、配角功能、高潮代價。
4. 最後才微調語氣與節奏。

硬性要求：
- 不可改變已確認的積分、物品位置、角色狀態、時間線，除非報告明確指出原文錯誤。
- 不可把 `pending_canon` 當成既成事實。
- 不可把便利店規則直接套到新副本。
- 不可用作者旁白說明「這是伏筆」「第二章如何如何」。
- 把資訊改成：角色觀察、系統提示、行動結果、對話暗示。

品質要求：
- 保留原章最強的爽點或恐怖點。
- 低分項要至少修補兩項。
- 章末鉤子要具體：人物、物品、規則、票根、倒數、照片、聲音、身份欄等至少一個發生變化。
- 如果高潮成功，必須留下後果或新疑問。

輸出：
- 輸出修訂後的完整 Markdown 章節。
- 不要附加解釋。
```
