# 伏筆回收 Prompt

> 用途：把早前埋下的物件、台詞、規則、異常行為，轉化為後文的解法、反轉或新危機。  
> 適用：卷結、章末反轉、系統補錄、身份揭示、道具用途揭露。

---

## Prompt

```text
請根據目前章節與伏筆管理，設計一個自然的伏筆回收方案。

回收原則：
1. 伏筆回收必須讓讀者回想前文時覺得「原來如此」，不是作者硬補。
2. 回收不能只是解釋設定，必須改變當前局面：通關、失敗、代價、身份改變、規則改寫、新危機。
3. 回收時避免直接說「這是伏筆」。讓角色透過觀察、對話、系統提示或物品變化理解。
4. 一次只回收一層，不要把全部真相一次講完。
5. 回收後必須留下下一層疑問。

請輸出：
1. 可回收伏筆清單。
2. 最適合本章回收的 1–2 個伏筆。
3. 每個伏筆的前文證據。
4. 回收方式。
5. 回收後的新疑問。
6. 可直接插入正文的段落。

禁止：
- 不要讓系統替作者完整解釋真相。
- 不要用回憶大段複述前文。
- 不要讓角色突然知道自己不可能知道的資訊。
- 不要為了回收而改變前文已確認事實。
```
