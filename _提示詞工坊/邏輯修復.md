# 邏輯修復 Prompt

> 用途：章節中出現硬邏輯問題，例如交易不成立、物品位置錯亂、角色行動來源不清、系統判定牽強。  
> 適用：讀者會問「為什麼能這樣？」或「這不是硬湊嗎？」的段落。

---

## Prompt

```text
請修復以下章節段落的硬邏輯問題。

修復原則：
1. 先列出目前段落的「事實鏈」：誰、在何處、持有什麼、做了什麼、系統如何判定。
2. 找出讀者會卡住的斷點。
3. 不要用作者解釋硬補；必須用角色觀察、系統提示、物品狀態、對話或行動補上因果。
4. 如果某個設計本身牽強，請改成更自然的同類型設計，而不是繼續硬拗。
5. 系統判定必須冷酷但可回推，不能為了讓主角通關而臨時改規則。

必查項：
- 物品是否真的在角色手上？
- 系統判定是否符合前文規則？
- 交易是否真的有買方、物品、付款 / 退貨 / 回收 / 抵押？
- 配角提示是否符合其已知立場？
- 是否有「作者知道，但角色不可能知道」的句子？

輸出：
1. 問題診斷，最多 5 點。
2. 修復方案，最多 3 個。
3. 推薦方案。
4. 修訂後段落。
```
