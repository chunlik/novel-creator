# 爽點加強 Prompt

> 用途：章節邏輯已成立，但讀起來不夠爽、不夠有反應、不夠有代價。  
> 適用：規則流解法、交易成功、身份反打、系統漏洞利用、伏筆回收。

---

## Prompt

```text
請在不改變劇情事實的前提下，加強本章爽點。

加強方向：
1. 找出主角最聰明、最反打、最利用規則漏洞的一刻。
2. 在該處前面補 1–2 個短句，讓壓力更明確。
3. 在該處後面補 1–3 個反應：彈幕停頓、配角表情、系統短訊、敵人動作、環境變化。
4. 爽點不要解釋過多，讓讀者自己看懂「原來可以這樣」。
5. 爽點後必須留一個陰影：代價、倒數、污染、記憶缺口、配角異常反應、系統扣分。

禁止：
- 不要讓主角突然變中二或大聲宣言。
- 不要讓配角集體吹捧主角。
- 不要把系統漏洞寫成萬能外掛。
- 不要改變已確認的交易結果、物品位置、積分數值。

輸出：
- 只輸出需要替換的段落，並標明建議插入位置。
```
