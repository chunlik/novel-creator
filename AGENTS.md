# AI Agent Operating Protocol

本 repo 是一套 AI 長篇小說創作系統。任何 AI agent 進入本 repo 時，必須先讀本檔，再操作腳本或修改內容。

## 最高原則

1. **不要只產生空範本。** 如果任務是封卷、開卷、寫總結、建立啟動包，AI agent 必須根據 repo 現有內容自動補完判斷型欄位。
2. **不要留下 TODO，除非資料真的不足。** 若資料不足，使用 `UNKNOWN:` 說明缺失來源，不要編造。
3. **章節正文維持簡體中文。** 系統文件、流程文件可用繁體中文；小說正文、章節標題、角色語氣依既有作品規範使用簡體中文。
4. **優先保護連續性。** 角色狀態、物品狀態、伏筆狀態比單章漂亮句子更重要。
5. **所有跨卷操作都要先封卷，再開卷。** 不可跳過卷總結直接寫下一卷。

## 權威資料來源順序

AI agent 判斷故事狀態時，依下列順序採信：

1. `無限直播間/03-章節/`：已完成章節，是最高敘事事實。
2. `無限直播間/04-狀態追蹤/角色狀態機.md`：角色約束與當前狀態。
3. `無限直播間/04-狀態追蹤/物品連續性.md`：物品位置、持有者、消耗狀態。
4. `無限直播間/04-狀態追蹤/伏筆管理.md`：伏筆 active / paid 狀態。
5. `無限直播間/04-狀態追蹤/全局摘要.md` 與 `時間線.md`：快速摘要與時間順序。
6. `無限直播間/02-大綱/`：計畫與方向；若與已寫章節衝突，以章節為準。
7. `無限直播間/05-AI上下文/`：當前工作包與限制。
8. `_共用範本/`：格式與檢查表。

## 標準任務：封卷

當使用者說「第一卷寫完了」、「幫我封卷」、「finish volume 1」等，AI agent 應執行：

```bash
python _腳本/check_consistency.py --novel infinite_livestream --volume {V}
python _腳本/close_volume.py --novel infinite_livestream --volume {V} --title "{卷名}" --overwrite
```

然後必須自動打開並補完：

```text
無限直播間/02-大綱/卷/{V}-{卷名}_卷總結.md
```

補完內容時必須：

- 讀取本卷所有章節。
- 整理本卷一句話總結。
- 列出本卷核心變化。
- 判斷本卷完成事項。
- 根據 `伏筆管理.md` 區分已回收與仍需承接。
- 根據 `角色狀態機.md` 補角色封卷狀態。
- 根據 `物品連續性.md` 補物品封卷狀態。
- 明確寫出下一卷必須承接事項。
- 移除可被合理補完的 `TODO`。

## 標準任務：開新卷

當使用者說「開始第二卷」、「open volume 2」、「start next volume」等，AI agent 應執行：

```bash
python _腳本/start_volume.py --novel infinite_livestream --volume {V} --title "{新卷名}" --first-chapter {N} --overwrite
```

然後必須自動補完：

```text
無限直播間/02-大綱/卷/{V}-{新卷名}.md
無限直播間/05-AI上下文/卷{V}-{新卷名}_啟動包.md
無限直播間/05-AI上下文/下一章指令.md
```

補完內容時必須：

- 讀取前一卷卷總結。
- 讀取角色狀態機、物品連續性、伏筆管理。
- 根據前卷結尾設計新卷開場狀態。
- 規劃新卷第一章必須承接的身體狀態、物品狀態與主線疑問。
- 設計新卷表面規則與真正陷阱。
- 決定哪些伏筆本卷推進、哪些只暗示不揭露。
- 移除可被合理補完的 `TODO`。

## 標準任務：寫下一章

當使用者要求寫下一章，AI agent 應先產生上下文：

```bash
python _腳本/compile_context.py --novel infinite_livestream --chapter {N} --vector-query "{本章核心線索}"
```

然後根據輸出內容寫章節。章節必須：

- 使用 `_共用範本/章節範本.md` 的 frontmatter 結構。
- 放入 `無限直播間/03-章節/{volume_dir}/`。
- 遵守 `05-AI上下文/禁止事項.md` 與 `風格規則.md`。
- 不主動解釋世界觀。
- 不破壞角色狀態機、物品連續性、伏筆狀態。

寫完後執行：

```bash
python _腳本/update_state.py --novel infinite_livestream --volume {V} --chapter {N} --summary "..." --timeline "..."
python _腳本/check_consistency.py --novel infinite_livestream --volume {V} --chapter {N}
```

再根據章節內容更新：

- `角色狀態機.md`
- `物品連續性.md`
- `伏筆管理.md`

## AI 不可做的事

- 不可因為範本有 TODO 就把 TODO 原樣交回給使用者。
- 不可在沒有章節證據時編造已發生事件。
- 不可把 active 伏筆擅自改成 paid，除非章節中已有明確回收。
- 不可讓已消耗物品無交代重新出現。
- 不可讓角色限制無交代解除，例如許小小 3 米限制、周仓心跳倒數代價。
- 不可在章節正文使用台灣式引號「」。章節正文使用 `""`。

## 輸出格式

完成任務後，AI agent 應回報：

```text
完成：
- 修改了哪些檔案
- 執行了哪些檢查
- 還有哪些 UNKNOWN 或需要作者決策的地方
- 下一步建議指令
```

若檢查失敗，必須先修正可自動修正的問題，再回報剩餘問題。
