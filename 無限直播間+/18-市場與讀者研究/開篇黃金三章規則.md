# 開篇黃金三章規則

## 文件狀態

【已定案】

本文定義《無限直播間+》開篇前三章的黃金規則——讀者在前三章中決定是否追讀，因此這三章的結構、資訊投放、鉤子安排至關重要。

本文不重複 [[行文風格規範]] 的句子級規則——本文專注於**章節級結構設計**。

---

## 核心原則

【已定案】

> 前三章的目標不是「講完開頭」——是「讓讀者想讀第四章」。

三個問題必須在前三章被回答：

1. **這是什麼世界？**（讀者理解基調——不一定理解所有規則，但理解「規則不一樣」）
2. **主角是誰？**（讀者關心周倉——他與眾不同、他有弱點、他有理由行動）
3. **為什麼繼續讀？**（讀者有一個或多個「我想知道接下來會怎樣」的問題）

---

## 第一章規則

### 第一章的任務

| 任務 | 說明 |
|---|---|
| **立刻進入異常** | 不從日常開始。第一段就是異常狀態。 |
| **建立周倉** | 名字、處境、第一個記憶點（讓讀者記住他是誰） |
| **建立核心機制** | 讀者理解「這是直播間／規則怪談」——不一定要理解所有規則 |
| **第一個衝突** | 不是「未來會有危險」——是「現在就有危險」 |
| **章末強鉤子** | 五種有效類型之一（見 [[行文風格規範]]） |

### 第一章禁止做的事

| 禁止 | 原因 |
|---|---|
| 從日常開始（「周倉是一個普通的大學生…」） | 讀者在前 10 秒就會關掉 |
| 前 500 字沒有衝突 | 讀者的 30 秒決策窗口中必須出現衝突 |
| 大量世界觀說明 | 讀者還沒關心周倉，不會有耐心讀設定 |
| 周倉的內心獨白超過 3 句 | 第一章的節奏必須快——行動 > 思考 |
| 章末沒有鉤子 | 讀者沒有理由點第二章 |

### 第一章的理想結構

```
段落 1-3（前 300 字）：
  → 異常狀態。周倉在異常中。讀者立刻知道「不對勁」。
  → 周倉的名字出現。
  → 第一個感官細節（不是抽象形容）。

段落 4-8（300-800 字）：
  → 第一個衝突出現（小衝突——不是終局危機）。
  → 讀者開始理解「這是什麼類型的異常」（系統？規則？怪物？）

段落 9-14（800-1800 字）：
  → 衝突升級（周倉主動做了一個決定）。
  → 第一個規則被揭示（假規則或真規則）。
  → 讀者第一次看到周倉的智力／觀察力。

段落 15-20（1800-2500 字）：
  → 衝突的高潮或轉折。
  → 章末鉤子——不是「接下來會發生什麼」，是「我必須知道接下來會發生什麼」。
```

## 第二章規則

### 第二章的任務

| 任務 | 說明 |
|---|---|
| **世界規則擴展** | 讀者得到更多關於「這個世界如何運作」的資訊——不是透過說明，而是透過情節 |
| **周倉的與眾不同** | 讀者看到周倉和其他人不一樣——他的觀察力、他的冷靜、他的異常 |
| **代價出現** | 第一章的衝突產生後果。試探規則的人付出代價 |
| **第二個鉤子** | 比第一章的鉤子更深——不是「會發生什麼」，是「這一切的真相是什麼」 |

### 第二章禁止做的事

| 禁止 | 原因 |
|---|---|
| 資訊倒退（重複讀者已知的內容） | 浪費第二章的資訊增量窗口 |
| 周倉被動（整章在消化第一章的後果） | 周倉必須在第二章有主動行動 |
| 規則仍是模糊的 | 第二章結束時，讀者應該對「規則的大致方向」有理解 |

### 第二章的理想結構

```
段落 1-4（前 500 字）：
  → 承接第一章的鉤子（不要跳時間）。
  → 周倉的即時反應——不是思考，是行動。

段落 5-12（500-1500 字）：
  → 第一個規則的測試——周倉用一個小動作試探邊界。
  → 讀者看到「試探的代價」。
  → 其他角色的反應（彈幕、其他主播、副本人物）提供不同視角。

段落 13-18（1500-2500 字）：
  → 新資訊：一條新規則、一個新異常、或一個新角色。
  → 周倉做出一個基於新資訊的決定。
  → 章末鉤子——比第一章更深，暗示「規則不是看起來那樣」。
```

## 第三章規則

### 第三章的任務

| 任務 | 說明 |
|---|---|
| **建立長期目標** | 讀者理解這本書不只是「一個副本」——有更大的故事在後面 |
| **第一個世界觀層面** | 一個小揭示——不是完整真相，是「這世界比讀者想的更大」的瞬間 |
| **情感線的第一次暗示** | 如果是許小小第一次出現／被暗示，必須讓讀者記住她 |
| **追讀理由確立** | 第三章結束時，讀者明確知道「為什麼我要繼續讀」 |

### 第三章禁止做的事

| 禁止 | 原因 |
|---|---|
| 第三章沒有任何新資訊 | 這是最常見的第三章失敗——重複前兩章的節奏而沒有升級 |
| 第三章結尾是「自然收尾」 | 前三章沒有「自然收尾」的權利——每一章結尾都是鉤子 |
| 全部規則已經揭示 | 第三章不應該揭示所有規則——留給後續章節 |

### 第三章的理想結構

```
段落 1-4（前 500 字）：
  → 承接第二章的鉤子。
  → 衝突升級——讀者意識到「這比表面更嚴重」。

段落 5-12（500-1500 字）：
  → 周倉理解到規則的一層真相（不是全部——是一層）。
  → 讀者獲得第一個「原來如此」瞬間。
  → 代價升級——讀者看到「在這個世界犯錯的後果」。

段落 13-18（1500-2500 字）：
  → 一個暗示——更大世界的存在（不是說明，是一個細節）。
  → 情感線的暗示（如果許小小已出現）。
  → 章末鉤子——讓讀者想追的不只是「下章」，而是「這本書」。
```

## 《無限直播間+》現有設計對照檢查

以下將黃金三章規則與現有的 階段一-卷一 設計進行對照：

### 現有章節拆分（來自 [[階段一-卷一-夜班便利店]]）

| 章節 | 現有功能 | 章末鉤子 |
|---|---|---|
| Ch.01 | 周倉進入直播間與便利店 | 螢幕顯示副本開始 |
| Ch.02 | 初步規則與第一波異常 | 紅雨衣顧客進店 |
| Ch.03 | 周倉發現規則矛盾 | 監控畫面出現不一致 |

### 第一章（Ch.01）對照檢查

| 黃金規則 | 現狀 | 判定 | 建議 |
|---|---|---|---|
| 第一段進入異常 | 「周倉進入直播間與便利店」 | ⚠️ 需確認 | 確保第一段不是「周倉在下班路上…」→ 第一句就應該在便利店裡，系統已啟動 |
| 周倉名字出現在前 100 字 | 待定 | ⚠️ 需確認 | 正文寫作時確認 |
| 前 500 字有衝突 | 「進入直播間」可能偏慢 | ⚠️ 需確認 | 考慮讓副本的第一條規則在前 500 字內彈出。不是「周倉發現自己在直播間」→ 是「系統彈出第一條規則，周倉來不及思考，因為紅雨衣已經進店了」 |
| 章末鉤子 | 「螢幕顯示副本開始」 | 🔴 偏弱 | 「副本開始」不是鉤子——讀者已經知道副本開始了。建議改成：① 第一條規則被周倉注意到某個不協調的地方；② 紅雨衣顧客第一次出現；③ 彈幕中出現一句不該出現的話 |
| 周倉的第一個記憶點 | 待定 | ⚠️ 需確認 | 周倉的第一個記憶點不應該是「他是 F-0217」——讀者還不知道這意味什麼。記憶點應該是行為：他冷靜、他注意到別人沒注意到的細節、他說了一句很乾的話 |

### 第二章（Ch.02）對照檢查

| 黃金規則 | 現狀 | 判定 | 建議 |
|---|---|---|---|
| 周倉主動試探 | 「初步規則與第一波異常」 | ⚠️ 待確認 | 確保周倉在第二章有主動行動——不是被動接收規則 |
| 代價出現 | 未知 | ⚠️ 需確認 | 建議：周倉試探第一條規則 → 付出小代價（不是大傷——是一個具體的感官後果） |
| 章末鉤子 | 「紅雨衣顧客進店」 | ✅ 夠強 | 紅雨衣是視覺記憶點 + 恐怖元素。確保進店的描寫有具體感官細節 |

### 第三章（Ch.03）對照檢查

| 黃金規則 | 現狀 | 判定 | 建議 |
|---|---|---|---|
| 第一個「原來如此」 | 「周倉發現規則矛盾」 | ✅ 方向正確 | 確保矛盾具體——不是「規則有點怪」，是「規則二的後半句和監控畫面對不上」 |
| 長期目標暗示 | 待定 | 🔴 缺失 | 第三章結束時，讀者需要一個「這不只是便利店的問題」的暗示。不是完整揭示，是一個細節：值班表上有不該存在的名字／商品標籤暗示人的身份被物化 |
| 許小小的第一次出現 | 現有設計：Ch.04 才出現 | ✅ 已定案修正 | **已決定採用方案 A**：許小小在 Ch.01 就以副本人物（女店員）出現。三層設計：① 表層——普通女店員 NPC；② 中層——細微不對勁（做 NPC 不該做的動作）；③ 底層——存續人格（Ch.01 不揭示）。見下方「最終決定」 |

### ⚠️ 關鍵問題：許小小在前三章的缺失（已解決）

~~現有章節拆分中，許小小在 Ch.04 才出現（「許小小給出關鍵提醒」）。這違反黃金三章原則。~~

**已定案——方案 A**。許小小在 Ch.01 就以副本人物（女店員）出現。

**許小小的 Ch.01 三層設計**：

| 層級 | 設計 | 讀者感受 |
|---|---|---|
| **表層**（看起來像） | 普通的便利店女店員。正在值夜班。對周倉的出現沒有特別反應——她也是這個副本的「一部分」 | 「又一個 NPC。」 |
| **中層**（細微的不對勁） | 她做了一個不屬於 NPC 的動作——比如看了一眼彈幕的方向（副本人物不應該能看到彈幕）、她在周倉說出某句話之前就已經在搖頭、她整理牛奶盒的方式和周倉的習慣一模一樣 | 「等一下——她怪怪的。」 |
| **底層**（Ch.01 不揭示） | 她是存續人格。她不是這個副本的原生 NPC——她被困在副本裡，和副本融為一體 | 留給後續揭示 |

**對章節拆分的主要影響**：
- Ch.01：新增許小小作為女店員的登場（簡短——一個段落即可。重點是讓讀者記得「有一個女店員」）
- Ch.04：許小小給出關鍵提醒——此功能保留。Ch.01 只是建立她的存在感，Ch.04 才是她第一次展現「她不只是 NPC」
- 牛奶盒這個記憶錨點可以在 Ch.01 就埋下（她在整理牛奶盒——這不是隨機動作）

### 被捨棄的方案

| 方案 | 被捨棄原因 |
|---|---|
| B：間接痕跡 | 痕跡不夠有記憶點——讀者對「具體的人」的記憶 > 「名字/彈幕」的記憶 |
| C：重組章節 | 不需要重組——Ch.01 加一段她的登場即可，不影響現有結構 |

## 十、平台版本開篇調整決策樹

【已定案】

### 10.1 不同平台版本的開篇調整決策樹

```
平台選擇 → 開篇策略

起點首發（方案 A — 預設推薦）
├─ 章節長度：2500-4000 字／章
├─ 第一章鉤子：規則矛盾 + 紅雨衣 / 不協調的細節
├─ 許小小出現：Ch.01 即出現（女店員，方案 A）
├─ 付費牆（20-30 章）前的爆點：第一個副本完整結束
└─ 哲學暗示：細節中暗示，不明說

番茄首發（方案 B — 需調整）
├─ 章節長度：2000-2500 字／章
├─ 第一章鉤子：極強爽點 + 高衝突
├─ 許小小出現：可延後到 Ch.02-03（番茄讀者對情感線耐心低）
├─ 付費牆（無）：前 10 章爽點密度極高
└─ 哲學暗示：大幅削減，只在細節中保留

飛盧首發（不推薦 — 讀者群不對）
├─ 章節長度：1500-2000 字／章
├─ 第一章鉤子：裝逼 / 反打臉
├─ 許小小出現：可在 Ch.05 後（飛盧讀者跳讀情感）
├─ 付費牆：第 1 章即收費
└─ 哲學暗示：不適用
```
