> **原則：** 場景機制不能跨場景直接套用。便利店、休息艙、記憶放映廳必須各自有自己的規則與限制。

- **副本編號：** C-0217

## 休息艙 / 過渡空間

### 當前狀態

- **狀態：** active
- **首次出現：** 卷1·Ch8
- **功能：** 副本結算後緩衝、下次進入倒數、系統訊息顯示。
- **危險性：** 不是安全區；直播圖標仍亮。

### 已建立機制

| 機制 | 說明 | 狀態 | 注意 |
|------|------|------|------|
| 結算後切換 | 周仓從便利店切入休息艙 | active | 不是正常移動，是場景替換 |
| 屏幕 / 終端 | 顯示歡迎、訊息、倒數 | active | 可用於商城、訊息、規則提示，但不應解釋過多 |
| 倒數 | 約三天後進入下一副本 | active | 可被提前切斷，但需正文提示 |
| 直播圖標 | 仍亮 | active | 表示被觀看狀態延續 |
| 陳述訊息 | 可收到外部 / 前任留下的訊息 | active | 訊息短，不能過度說明真相 |

### 未確定機制

| 機制 | 狀態 | 使用方式 |
|------|------|----------|
| 是否有門 | unknown | 不可直接寫普通房門；若要有出口，需系統或空間異常呈現 |
| 是否可購買道具 | likely_but_unconfirmed | 可由終端正式顯示商城後使用 |
| 是否有其他人 | unknown | 他人出現需有系統、直播、關聯實體或空間異常邏輯 |
| 許小小是否在場 | pending_canon | 若保留關聯實體設定，第9章需正式落地 |

### 禁止錯誤

- 不可把休息艙寫成現實世界房間。
- 不可讓周仓出門買東西、回家、上班。
- 不可讓直播無交代關閉。
- 不可讓休息倒數無交代消失。
- 不可把商城道具寫成已自動發放。

## 記憶放映廳（卷2副本）

### 當前狀態

- **狀態：** planning
- **首次正式進入：** 預計卷2·Ch9
- **功能定位：** 記憶 / 身份 / 放映 / 歸檔恐怖。
- **核心危險：** 畫面是真的，但被剪掉的空白才是陷阱。

### 預定機制

| 機制 | 狀態 | 建議落地方式 |
|------|------|--------------|
| 票根 / 座位號 | planning | 第9章發放或出現在周仓手中 |
| 觀眾席 | planning | 不可回頭；可用聲音、反光、空位變化暗示 |
| 銀幕 / 放映 | planning | 播放真實但被剪輯過的記錄 |
| 字幕 | planning | 與畫面不一致時形成陷阱 |
| 回答問題 / 提交片名 | planning | 可連接交易 / 歸檔 / 記憶扣除 |
| 本人確認 | planning | 若畫面中出現自己，不能承認或確認 |
| 記憶扣除 | planning | 答錯或違規時失去某段細節，而非單純受傷 |

### 與卷1的連接方式

記憶放映廳可以使用卷1元素，但只能透過：

- 回放畫面
- 被剪輯的記錄
- 票根 / 座位 / 片名
- 字幕污染
- 觀眾席反應
- 系統歸檔資料

不可直接使用：

- 便利店完整空間
- 黑水實體
- 收銀台交易流程
- 便利店燈光庇護區
- 紅雨衣 15 分鐘巡邏規則

除非正文正式建立「放映廳正在播放便利店記錄」。

### 禁止錯誤

- 不可把記憶放映廳寫成普通電影院。
- 不可讓它只有恐怖氣氛而沒有系統機制。
- 不可一次展示全部規則。
- 不可讓觀眾席直接解釋彈幕本質。
- 不可讓照片完整揭露所有真相。

## 第二卷寫作檢查

寫卷2時，每章至少檢查：

- 本章場景是否明確？
- 有沒有把休息艙寫成現實？
- 有沒有把便利店機制直接搬到放映廳？
- 新副本機制是否在使用前已被讀者看到？
- 場景切換是否有系統、倒數、光線、銀幕、門、票根等媒介？
- 角色 / 物品是否跟著場景合理移動？
