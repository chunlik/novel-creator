# AI 導航索引卡

> **使用規則：** 接到任務 → 查下方對應行 → 只讀「必讀」欄 + 按需讀「參考」欄 → 執行 → 更新「事後更新」欄。

> **AI 輸出格式：** 每次回覆開頭顯示當前流程狀態。格式為 `章節流程 [Step X/8]` 或 `大綱流程 [Step X/6]` 加簡短動作說明。非流程進行中的回覆則不加。

## 一、權威資料來源（精簡 4 層）

寫作時判斷故事事實的採信順序：

| 優先級 | 來源 | 說明 |
|--------|------|------|
| **L1 最高** | `03-章節/` 已寫章節 | 已完成的正文是最高敘事事實 |
| **L2** | `04-狀態追蹤/` 角色狀態機 + 物品連續性 + 伏筆管理 + 全局摘要 + 時間線 + 系統狀態 + 規則連續性 | 跨章約束與當前狀態 |
| **L3** | `02-大綱/` 卷大綱 + 伏筆地圖 + 角色弧線 + 張力曲線 + 副本設計 | 計畫與方向；若與已寫章節衝突，以章節為準 |
| **L4 參考** | `05-AI上下文/` 當前寫作包 + 下一章指令 + 風格規則 + 禁止事項 | 當前工作包與寫作限制 |
| **L4 參考** | `06-知識庫/` | 寫作參考資料，不直接影響事實判斷 |

## 二、寫作流程（最終確認版）

### 有大綱時的章節流程

```
Step 1 — Read 前置上下文（批次讀）
├── 卷大綱（本章目標+事件）
│   └── 02-大綱/卷/卷{V}-{卷名}/01-卷大綱.md
├── 05-AI上下文/
│   ├── 當前寫作包.md
│   └── 下一章指令.md
├── 風格規則.md + 禁止事項.md
├── 04-狀態追蹤/
│   ├── 角色狀態機.md
│   ├── 物品連續性.md
│   ├── 伏筆管理.md
│   ├── 全局摘要.md
│   └── 時間線.md
└── 章節範本
    └── _共用範本/章節範本.md

Step 2 — Run 腳本
└── python _腳本/compile_context.py --novel {novel_id} --chapter {N}

Step 3 — Grill for chapter（使用 `/grill-me` 技能，每次提問附建議）
├── 載入 grill-me skill → 討論方向、鉤子、關鍵選擇、代價
├── AI 每問一題必須先給自己的建議選項
├── 在 grill 討論中確認所有關鍵決策
└── 確認後才進入 Step 4 寫章

Step 4 — Write chapter
├── 使用章節範本 frontmatter
├── 簡體中文 + "" 雙引號
├── 遵循風格規則（主語控制、異常不解釋等）
├── 寫完後必須對章節範本 checklist 逐項打勾確認（主語控制/雙層衝突/異常信號/章末鉤子/規則檢查/空間動作邏輯/推理分析驗證/敘述技法）
└── 放入 03-章節/{volume_dir}/

Step 5 — Check / Review / Proofread
├── 主語控制（連續不超過2句以"他"開頭）
├── 簡體中文（無繁體/台灣用詞）
├── 引號格式（"" 非 「」）
├── 因果鏈 + 空間一致性 + 數字準確
├── 邏輯驗證（每條規則有前因後果）
├── 章末鉤子有效性（動作中斷型/狀態反轉型優先）
└── python _腳本/check_consistency.py --novel {novel_id} --volume {V} --chapter {N}

Step 6 — Confirm（作者確認沒問題）

Step 7 — Update 所有必要文件
├── python _腳本/update_state.py --novel {novel_id} --volume {V} --chapter {N} --summary "..." --timeline "..."
├── 04-狀態追蹤/
│   ├── 角色狀態機.md（新約束、代價變化）
│   ├── 物品連續性.md（位置/持有者/消耗）
│   ├── 伏筆管理.md（新種=active，回收=paid）
│   ├── 全局摘要.md（新增一行摘要）
│   ├── 時間線.md（新增時間記錄）
│   ├── 系統狀態.md（積分/道具/倒數）
│   └── 規則連續性.md（規則變化/污染）
├── 按需更新：場景機制.md / 重大轉折.md / 謎團管理.md
└── 05-AI上下文/
    ├── 當前寫作包.md（更新進度）
    └── 下一章指令.md（更新為下一章要求）

Step 8 — Vol summary（如本卷完成）
└── 執行封卷流程
```

## 五、寫章後必更新的狀態檔（依順序執行）

> ⚠️ 每完成一章後，你必須**依序執行以下三步**，不可省略任一步。

### Step 1：執行腳本

```bash
python _腳本/update_state.py --novel infinite_livestream --volume {V} --chapter {N} --summary "一句話摘要" --timeline "時間點"
```

### Step 2：手動更新追蹤檔案

| 檔案 | 更新方式 |
|------|---------|
| `角色狀態機.md` | 在「狀態變化記錄」表格新增一行：新狀態、新約束、代價變化 |
| `物品連續性.md` | 更新相關物品的「當前位置」「狀態」「持有者」欄位 |
| `伏筆管理.md` | 新種伏筆 → 新增一行，status=active；已回收 → status=paid |
| `全局摘要.md` | 在摘要表新增一行：章節、一句話摘要 |
| `時間線.md` | 在時間表新增一行：時間點、事件 |
| `系統狀態.md` | 更新積分、道具狀態、倒數數值 |
| `規則連續性.md` | 任何規則變化（新發現、被污染、被打破）都要記錄 |

### Step 3：一致性檢查

```bash
python _腳本/check_consistency.py --novel infinite_livestream --volume {V} --chapter {N}
```

## 六、動態維護協議（防止檔案過期）

> 目標：追蹤檔案永遠不會落後於已寫章節。

### 6.1 寫章時同步更新

每次寫完一章，必須執行第五節的 Step 2（手動更新追蹤檔案）。
不可寫完章節但不更新追蹤檔——這會導致下一章 AI 基於過期資料寫作。

### 6.2 跨章回補檢測

如果你接手一個已有章節但追蹤檔可能落後的狀態：

1. 檢查 `04-狀態追蹤/全局摘要.md` 的最後記錄章節號。
2. 對照 `03-章節/{volume_dir}/` 的實際章節列表。
3. 如果缺少 N 章的記錄，你必須：
   a. 依序讀取所有未記錄的章節。
   b. 為每一章同步更新所有追蹤檔案。
   c. 在全局摘要開頭加註：「回補：第 X～Y 章」。

### 6.3 定期過期檢測（每寫完一卷執行一次）

封卷時，檢查以下檔案的最後更新時間是否與本章節相符：

| 檢查項 | 通過條件 |
|--------|---------|
| 全局摘要的章節數 = 實際章節數 | 無遺漏 |
| 角色狀態機有最後一章的記錄 | 角色約束已收束 |
| 物品連續性有最後一章的記錄 | 物品位置已鎖定 |
| 伏筆管理：本卷應回收的伏筆皆已 paid | 對照 02-伏筆地圖.md |
| 系統狀態的最後記錄 = 結算結果 | 積分/道具一致 |
| 規則連續性記錄了本卷所有規則變化 | 無遺漏 |

如果任何一項不通過，必須在封卷前補齊。

### 6.4 腳本輔助檢測

```bash
# 檢查一致性（含追蹤檔 vs 章節比對）
python _腳本/check_consistency.py --novel infinite_livestream --volume {V}
```

- [ ] 引號：全部使用 `""`（非 `「」`）？
