## C. 系統本質與直播間

已定：

1. 直播間來源採用三層反轉：前期像組織創造，中期揭示組織只是發現並改造，後期揭示直播間是歸零事件的篩選器官。
2. 組織改造了管理層與人性化偽裝，包括直播介面、主播編號、評級、積分、商城、休息艙、觀眾席、管理員流程。
3. 觀看功能是「觀測穩定 + 污染分攤」。
4. 主播本質是探針、穩定器、歸零篩選樣本、交易代理。
5. 主播拒絕副本會扣存在保證金，扣完後降格為低權重觀眾碎片。
6. 存在保證金來源是現實存在抵押 + 系統預支。
7. 積分本質是存在盈餘、觀測信用、格式化污染殘差。
8. 商城主要出售情報與權限。

仍未決：

1. 系統是否有自我意識？如果有，是原始系統自我意識，還是吸收人類管理層後長出來的？
2. 系統是否被黑水污染，還是黑水本來就是系統排泄物？
3. 組織的正式名稱、歷史、核心人物。
4. 管理層最初如何失控。
5. 休息艙與共享大廳的底層功能。

## G. 觀眾與彈幕

已定：

1. 觀眾來源是研究員 → 志願者／關聯者 → 失敗主播 → 混合碎片池。
2. 觀眾發彈幕有三重動機：被看見、轉嫁污染、累積殘留權重。
3. 觀眾池會互相吞噬，系統也會定期清理低權重碎片。
4. 系統會用被清理碎片生成偽彈幕，高權重觀眾也能冒充別人。
5. 高權重觀眾只能取得表層語料、情緒殘響與破碎名詞，不能完整理解因果。
6. 真偽彈幕判斷依靠私密記憶驗證 + 語義污染痕，但沒有絕對判定。
7. 打賞存在，但打賞不是錢，而是觀眾自己的殘留；打賞只能提供資訊。

仍未決：

1. 觀眾側是否有明確物理空間？
2. 觀眾碎片是否都知道自己已死／被抹消？
3. 周倉偽裝成觀眾時，最多能做到什麼程度？
4. 陳述是否已在觀眾側？
5. 觀眾池是否有階層、幫派或高權重觀眾勢力？

## K. 禁用與注意

1. 不使用許小小 3 米限制。
2. 不讓許小小第一 Part 直接離開便利店。
3. 不讓周倉直接改變過去。
4. 不把許小小寫成普通助手。

- 每次新設定加入時，要檢查是否削弱第一 Part 的情感鋪墊。

- 林█不能太早揭示身份。
- 系統真相不能太早說破。
