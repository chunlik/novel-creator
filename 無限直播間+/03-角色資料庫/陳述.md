## 角色功能

【已定案】

陳述承擔以下敘事功能：

- 作為周倉的**前路鏡像**——展示「想救一個人但用錯方法」的後果；
- 透過 F-0217 的汙染，為周倉提供 Part 1–2 的邊緣權限；
- 留下筆記、碎片與線索，成為周倉調查的動力與資訊來源；
- 他失敗的案例是周倉最重要的反面教材；
- 他的同化者狀態，展示「不完全抹消」的其中一種結局；
- 透過沈未提供的修復 X 舊檔案，將陳述／X／許小小三線接在一起。

## 歷史

【已定案】

### F-0217 時期

陳述曾是 F-0217 功能位的持有人。他利用 F-0217 的觀測權限追蹤 X 的殘留痕跡，試圖證明被系統刪除的人可以被找回。

他做了三件系統不允許的事：

| # | 行為 | 目的 |
|---|---|---|
| 1 | 非法記錄層存取 | 查找 X 的刪除前完整記錄 |
| 2 | 權限自我擴張 | 讓 F-0217 權限短暫擴張到鄰近權限域，擴大搜尋範圍 |
| 3 | 碎片錨點植入 | 逃離時將筆記片段與記憶殘留嵌入 F-0217 底層——不讓線索被系統清除 |

這三個行為永久汙染了 F-0217，後任者周倉自動繼承了這些偏移。

### 與 X 的關係

X 是陳述曾帶出的關聯實體——類似許小小之於周倉。

- 陳述曾相信副本裡的人可以被帶出去。
- 陳述曾成功把 X 帶出副本。
- X 之於陳述，類似許小小之於周倉。

### 失敗：回收本體方案

陳述發現 X 在離開副本後人格發生變化——她長出了新的自我。他不接受這個變化，申請恢復 X 的「本體連續性」。

[[保守存續派]] 將他的申請轉譯為「回收本體方案」，與系統核心合作執行：

1. 陳述申請恢復 X 的「本體連續性」。
2. 保守存續派將此申請轉換為「回收本體方案」。
3. 系統認定 X 的後生人格為不一致存在，啟動校正。
4. X 的後生人格被清除。
5. 本體 X 被回收為系統功能元件。
6. 陳述救援失敗。

陳述的錯誤不是「想救 X」——是**太想把她救回原本的樣子，忽略了她已經長成不同的人**。

> 陳述當年也說，他只是想把她找回來。可他找回來的，不一定是她想成為的那個人。
> — [[沈未]]

### 成為同化者

【已定案】

陳述失敗後，系統將他標記為「已結束」。但他的存續意志太強——系統無法完全抹消。

系統選擇將他轉為**同化者**：不再是一個完整的人，而是分散在多個系統節點中的碎片功能元件。

## 同化者狀態

【已定案】

陳述現在不是死，不是活，是被系統部分回收但拒絕完全解散。

### 當前存在形式

| 存在位置 | 表現形式 |
|---|---|
| F-0217 功能位底層 | 三個越權行為留下的永久偏移；碎片錨點中的記憶殘留 |
| 觀眾側 | 零星的「陳述碎片」——短暫逸出的殘留意識，可能出現在彈幕或觀眾池邊緣 |
| 記錄層殘留 | 未清理乾淨的舊查詢紀錄、筆記殘頁、觀測路徑 |
| 商城情報 | 陳述留下的異常註記與舊檔案 |

### 同化者的特性

- 陳述仍然存在，但不是作為一個能被對話或互動的獨立人格。
- 他的碎片殘留仍在**持續微調**他當年的後門——F-0217 中偶爾出現的新殘留，可能來自現在的陳述碎片試圖透過舊節點發訊號。
- 這也是系統無法清除 F-0217 汙染的原因之一：汙染源（陳述的功能化碎片）和 F-0217 之間存在持續的低活性連結。

### 陳述是否還想救 X

【已定案】

部分碎片表現出悔恨——他知道自己錯了。部分碎片仍在盲目的搜尋模式中——它們只剩下「找 X」這個指令。部分碎片在試圖警告周倉不要重蹈他的錯誤。

陳述不是一個統一意識。他是**碎片集合**——不同碎片知道不同的東西、有不同的情感狀態。

## 與周倉的關係

【已定案】

### 鏡像結構

| 陳述線 | 周倉線 |
|---|---|
| 陳述帶出 X | 周倉與許小小建立未完成交易 |
| 陳述想救回 X | 周倉想找回許小小 |
| 陳述差點抹掉 X 的新人格 | 周倉必須避免把許小小還原成 X |
| 陳述留下筆記與痕跡 | 周倉留下跨時間錨點痕跡 |
| 陳述失敗 | 周倉必須做出不同的選擇 |

### 頻率共振，不是轉世

周倉和陳述的連結不是血緣、不是轉世、不是「周倉就是陳述的後續身份」。

他們的連結是**頻率共振**：周倉的認知模式與陳述相似——觀察力、對系統的不信任、對被刪除者的同理心——因此 F-0217 選擇了周倉。

> 他的觀察方式像陳述，僅此而已。

### 核心差異

陳述想**還原**——把 X 變回最初的樣子。周倉的任務是**存續**——讓許小小作為許小小存在，而不是被還原成 X。

這就是為什麼周倉必須成功而陳述失敗：因為陳述的問題是錯的。問題不是「如何把人變回原樣」，而是「如何讓已經變成另一個人的人，繼續存在」。

## 設計原則

【已定案】

- 陳述**不直接登場**——他只透過筆記、碎片、殘留與他人轉述出現；
- 陳述的錯誤必須夠具體、夠合理——他犯的錯不是愚蠢，而是**人性中可以被理解的執念**；
- 陳述 vs 周倉的核心對比是「還原」vs「存續」——這不是善惡對立，是兩種答案；
- 陳述的同化者狀態必須保持殘缺——不能一次講完真相，不同碎片給出不同線索；
- 陳述不是「失敗的周倉」——周倉不是「成功的陳述」。他們是不同的路。

## 修改風險

【中】

陳述是 X／許小小／周倉長線的關鍵環節。修改陳述的失敗原因或同化者狀態，會影響許小小的終局情感回收、周倉的終局選擇、以及「修復 vs 抹殺」的核心答案。
