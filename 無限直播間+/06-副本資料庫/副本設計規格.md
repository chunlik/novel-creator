# 副本設計規格

## 文件狀態

【已定案】本文件是副本功能規格表，不新增 canon。表內內容未經作者細節審核前，不可視為已定案。

本文件以「階段」管理副本功能與真相切片。既有檔案若仍使用 Part 作索引，僅視為目前對應檔案／舊索引備註，不作主軸。

## 使用邊界

副本是真相切片，不是普通關卡。每個副本只主打一種恐怖或倫理壓力，並承擔對應階段的資訊功能、情感功能與世界觀功能。

新增副本名稱、章級事件、具體通關橋段、角色名、規則細節、獎懲細節、能力限制或核心答案前，需先經作者審核。

## 規格欄位

| 欄位 | 填寫規則 |
|---|---|
| 副本名稱 | 只填既有副本名。後續階段未配置副本時，填「功能位」。 |
| 所屬階段／目前對應 Part/卷 | 以階段為主鍵；Part/卷僅作目前對應檔案／舊索引備註。 |
| 表面規則 | 只寫功能級表象，不新增具體章級規則。 |
| 真規則 | 只寫此副本承載的真相方向，不揭露凍結答案。 |
| 核心衝突 | 寫單一恐怖或倫理壓力，不混入多個主壓力。 |
| 情感功能 | 寫對角色關係或讀者情緒的功能，不新增事件。 |
| 世界觀功能 | 寫此副本推進哪一層世界觀。 |
| 能力/權限揭露 | 寫可露出的能力、判定或權限輪廓，不補完整機制。 |
| 新資訊 | 寫本副本允許讀者新知道的事。 |
| 代價 | 寫功能級代價方向，不新增具體懲罰橋段。 |
| 主要角色 | 只列既有副本檔已出現角色。 |
| 組織痕跡 | 只列既有副本檔已提及組織或派別。 |
| 伏筆 | 只列既有副本檔已提及伏筆方向。 |
| 卷末鉤子 | 寫功能級鉤子，不新增章級事件。 |
| 不可揭露內容 | 對照凍結邊界，標明不可寫入的核心答案。 |
| 作者審核狀態 | 允許值：`待審核／已確認／需重討／廢案`。本表目前功能級套用統一使用 `待審核`。 |

## 不可越界內容

以下內容只能作邊界提醒，不得在本規格補入核心答案：

- 系統起源完整版
- 歸零事件完整版
- 0號樣本完整身份
- 林█自我刪除型真相
- X 與許小小終局答案

## 檢查規則

- 每次新增副本列，先確認副本名稱已存在於既有資料或已由作者明確批准。
- 每次細化階段 1 或階段 2，僅依既有副本檔補功能，不新增章級事件。
- 每次細化後續階段，只能填功能位，直到作者批准副本名稱。
- 每次調整目前對應檔案／舊索引，只能作排程或舊索引備註，不得改成主軸。
- 每次補入真規則、能力/權限揭露、新資訊與卷末鉤子，先對照不可越界內容。
