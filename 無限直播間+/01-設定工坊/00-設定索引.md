## 已確認大方向

1. 全書先用 **Part／大篇章** 管理，不先鎖死卷數。
2. 終局方向採用：**改造直播間**。
3. 第一 Part 中，許小小不離開便利店，但她救過周倉。
4. 周倉與許小小之間留下未完成交易／未結清歸屬。
5. 中段重大事件為：**未登記主播篇**。
6. 林█後期逐步揭示為許小小未來／重組形態。
7. X 是陳述曾帶出的關聯實體；許小小是 X 被抹消後形成的情感殘留之一。
8. 陳述當年失敗，是因為想救回 X，卻忽略 X 已長出新的獨立人格；系統利用此錯誤誘導他走向「回收本體」。
9. 直播間來源採用三層反轉：前期像組織創造，中期揭示組織只是發現並改造，後期揭示直播間是歸零事件的篩選器官。
10. 觀看功能採用「觀測穩定 + 污染分攤」。
11. 觀眾席是研究員、志願者／關聯者、失敗主播、同化者碎片、系統偽彈幕的混合池。
12. 打賞存在，但只能提供資訊，不能給道具或直接保護。
13. 主播本質是探針、穩定器、歸零篩選樣本、交易代理。
14. 積分本質是存在盈餘、觀測信用、格式化污染殘差的混合物。
15. 商城主要出售情報與權限；情報可能被管理員、同化者、系統核心或舊組織殘留污染。

## 明確禁用／修正

| 設定 | 狀態 |
|---|---|
| 許小小 3 米限制 | 禁用 |
| 第一 Part 直接帶許小小離開便利店 | 禁用 |
| 許小小被當成交易代價收走 | 修正：不是代價，而是提交歸屬權作清算擔保 |
| 周倉可直接改變過去 | 禁用 |
| 打賞提供實體道具／護盾／治療 | 禁用 |
| 商城作為普通裝備商店 | 禁用 |

## 後續工作流程

1. 先完成所有設定模組討論。
2. 再根據設定自然拆分 Part。
3. 再決定每個 Part 需要多少卷／副本。
4. 最後才進入正文重寫或續寫。
