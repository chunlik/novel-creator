# 行文風格規範

## 文件狀態

【已定案】

本文是《無限直播間+》的行文風格總規範。所有正文寫作、AI 輔助寫作、審稿與重寫，都必須參照本文。

本文不涉及世界觀設定。世界觀設定查閱 [[已定案設定總表]]。

---

## 核心原則

【已定案】

> 讀者不是來看你展示世界觀的。讀者是來看周倉在規則的縫隙裡求生的。

三個最高原則：

1. **動作推進，不解釋**：不要讓角色／旁白解釋世界觀。讓規則在衝突中被發現。
2. **身體先於心理**：先寫身體反應（手抖、耳鳴、視線模糊），再寫心理反應。恐怖來自感官。
3. **每一段都有資訊增量**：對話不是為了填充字數。每段對話要麼推進情節，要麼揭示角色，要麼埋下伏筆。

---

## 句子規則

【已定案】

### 長度

- 敘述句：15–40 字為主。
- 動作句：越短越好，極限壓縮。
- 對話：不超過 30 字／句。角色不能像在寫論文。
- 心理活動：限制在 3–4 句以內。不要讓周倉在腦內寫散文。

### 結構

- 少用「與其說是 X，不如說是 Y」結構——這是 AI 腔的典型特徵。
- 少用「在某種意義上」「從某個角度看」——這些是思維的拐杖。
- 段落開頭不要連續三句用相同主語（「周倉…」「他…」「周倉…」）。
- 環境描寫不是為了美——是為了提供資訊（空間結構、異常細節、時間線索）。

### 禁止的句子結構

| 禁止 | 原因 | 改寫方式 |
|---|---|---|
| 「他意識到…」「他終於明白…」 | AI 解釋腔——讀者自己會意識到 | 直接寫發生的事 |
| 「這意味著…」「也就是說…」 | 把潛台詞講成明台詞 | 刪掉，留白 |
| 「不知為何…」「說不上為什麼…」 | 廉價的懸念製造 | 用具體感官細節取代 |
| 「彷彿」「似乎」「隱約覺得」 | 過度模糊——偶爾可用，不能成習慣 | 確定的寫確定，不確定的寫觀察 |
| 段末總結句（「這就是…的真相」「一切才剛開始」） | AI 最愛做的事——替讀者總結 | 刪掉 |

## 對話風格

【已定案】

### 規則

- 對話必須像人說話，不能像設定的載體。
- 角色不能說出「根據系統規則第三條…」——這不是說話，是朗讀設定檔。
- 同一角色在不同情緒狀態下，句子結構要不同（緊張時更短、更碎）。
- 彈幕不是對話——彈幕是碎片。保持破碎、口語、不完整。

### 各角色對話基調

| 角色 | 基調 | 備註 |
|---|---|---|
| 周倉 | 短、直接、有觀察力。緊張時更短；放鬆時會帶一點乾澀的嘲諷 | 不耍帥、不廢話 |
| 許小小 | 柔但不弱。語句稍長於周倉，但不囉嗦。關鍵時刻一句話可以很重 | 情感重量不是靠話多 |
| 沈未 | 平淡、精確、不廢話。像在翻一本已經看膩的檔案 | 偶爾一句帶感情的破綻 |
| 小眠 | 破碎、跳躍、有時像自言自語 | 修復後的遺留 |
| 彈幕 | 破碎、口語、不完整。有時是髒話、表情、一個詞。不寫成完整段落 | 這是碎片，不是文章 |

### 對話禁止

- 禁止角色說出只有作者／讀者需要的資訊（「你應該知道，這個世界的規則其實是…」）
- 禁止對話變成設定說明會
- 禁止所有角色用相同節奏講話

## 恐怖感寫法

【已定案】

《無限直播間+》的恐怖不是跳嚇、不是血腥、不是怪物追逐。

恐怖來源：
1. **規則本身**——當你意識到規則的後果，比怪物更可怕
2. **存在的脆弱**——被抹消不是死亡，是你從來沒有存在過
3. **熟悉的變異**——系統 UI 上的異常字、彈幕中不該出現的名字、鏡子裡多一隻手

### 寫法規則

- **身體細節 > 抽象形容**。不要寫「恐怖的氣氛籠罩了房間」。寫「牆上的系統字開始從右下角往上融化。不是字體特效——像蠟燭燒到一半的那種塌。」
- **一次只一個異常**。不要堆疊恐怖元素。一個細節夠好的異常，比五個同時出現的怪物更恐怖。
- **恐怖來自限制**——不要讓周倉能立刻理解一切。他看到縫隙、殘影、不該存在的物品，但他不知道那是什麼。限制 = 恐懼。
- **日常中的異常**。休息艙終端正常顯示主播資訊——但周倉的資訊欄有一個欄位是空的，而其他欄位在慢慢補全這個空位。

### 禁止的恐怖寫法

- 禁止「詭異」「恐怖」「毛骨悚然」等抽象形容詞——用感官細節代替。
- 禁止怪物登場時長篇外貌描寫——讀者會跳過。用一個最異常的身體細節破題。
- 禁止周倉在恐怖場景中做心理總結（「他終於明白這是怎麼回事了」）。讓他行動。

## 規則怪談專屬風格

【已定案】

規則怪談的核心閱讀快感：**看到規則→質疑規則→測試規則→發現假規則→推導真規則→為真規則付出代價**。

### 規則揭示節奏

| 階段 | 寫法 | 字數配比 |
|---|---|---|
| 規則呈現 | 清楚、簡短、像系統提示。不要修飾。規則本身夠恐怖 | 5% |
| 周倉的觀察 | 他注意到什麼矛盾、什麼漏洞、什麼不該出現的東西 | 15% |
| 試探 | 行動——不是思考。用一個小動作測試規則 | 20% |
| 代價 | 試探的後果。這是恐怖的峰值。用身體細節寫 | 25% |
| 真規則浮現 | 周倉／讀者開始拼出真正的規則。不是完整答案，是一個新的、更窄的理解 | 20% |
| 新行動 | 用新理解做一個決定。章末落在這個決定上 | 15% |

### 規則文字的寫法

- 系統規則用獨立段落、粗體或區塊呈現。
- 規則文字必須簡短、不修飾、像真正的系統提示：「禁止回頭」比「請注意，在任何情況下不得回頭查看」好一百倍。
- 假規則和真規則的文字風格要有微妙差異——假規則更工整，真規則更破碎。

## 直播間敘事專屬風格

【已定案】

### 彈幕寫法

- 彈幕不是旁白。彈幕是其他觀眾的碎片發言。
- 保持破碎：有時是一個詞、一個表情、一句髒話。
- 彈幕之間不互相解釋。它們是混亂的、矛盾的。
- 關鍵彈幕（來自許小小／陳述／高權重觀眾）要有一點不協調感——語氣、內容、時間點和一般彈幕不同。
- 系統偽彈幕的特徵要融入文本細節（太完整、太急切、目的太單一、不會承認不知道）。

### 直播間副本進入／退出

- 不要每次都寫完整的「系統提示→選擇→進入→場景描寫」流水帳。
- 第一副本詳細寫；後續副本只在有變化時寫差異。
- 退出不要每次都寫獎勵結算。只在關鍵節點（獲得重要道具／資訊／代價）時寫。

## 去 AI 味規則

【已定案】

這是最重要的規則。AI 輔助寫作的最大敵人不是文筆差，是「看起來像小說但讀起來不是」。

### 必刪清單（審稿階段單獨跑一輪）

| AI 腔特徵 | 處理 |
|---|---|
| 「他意識到…」「他終於明白…」「他突然想到…」 | 刪掉開頭，直接寫他明白的內容，或改成行動 |
| 「這讓他想起了…」 | 刪掉。如果回憶重要，用感官觸發（氣味、溫度、一句話），而不是這個句型 |
| 「不知為何…」「說不上為什麼…」 | 改成具體觀察。如果周倉不知道原因，就寫他觀察到的現象 |
| 「在某種意義上…」 | 刪掉整句 |
| 「既像…又像…」這種修辭堆疊 | 留一個，刪其他 |
| 段末總結句／警句 | 刪掉。讓讀者自己感受 |
| 「然而…」「但是…」開頭的轉折句（連續使用） | 每章限制在 3 次以內 |
| 對偶句、排比句、修辭過度完整的句子 | 口語感優先。不要每一段都像在寫作文 |
| 「一切的答案，都在…」類型的故作懸念 | 刪掉。真正的懸念不需要宣告 |

### 去 AI 味重寫原則

1. **縮短**：AI 寫的句子的第一個版本，通常可以刪掉 30% 的詞而不丟失資訊。
2. **打碎**：AI 喜歡四平八穩的結構。把一段拆成兩三段。留一個不完整的句子。
3. **加入感官**：AI 傾向寫「發生什麼」，缺少感官。補上溫度、聲音、氣味、身體反應。
4. **刪掉解釋**：AI 最大的問題是怕讀者不懂。相信讀者的推理能力。

## 章末鉤子規則

【已定案】

每章結尾必須讓讀者想點下一章。不允許「自然收尾」。

### 五種有效鉤子

| 鉤子類型 | 說明 | 範例 |
|---|---|---|
| **新規則暴露** | 發現一條新規則，或舊規則被推翻 | 「第三條規則的後半句，用的是過去式。」 |
| **代價揭露** | 某個行動的真正代價剛被發現 | 周倉低頭看手。無名指消失了。不是傷口——是指頭從來沒存在過。 |
| **身份反轉** | 某人的身份被質疑或逆轉 | 「你不是第一個 F-0217 來這裡的人。」 |
| **行動倒數** | 主角必須立即行動，章末落在倒數的起點 | 終端上浮出一行字：定位完成度 97%。 |
| **情感重擊** | 一個情感真相（通常是許小小相關） | 彈幕飄過一行：她問你還記不記得牛奶盒放在哪裡。這不是周倉記憶中的那個人問的。 |

### 禁止的章末

- 「他走進了下一個房間。」不是鉤子。
- 「一切才剛開始。」是廢話。
- 「但他不知道的是…」是AI腔。
- 「接下來發生的事，將改變一切。」是廢話中的廢話。

## 信息增量硬規則

【已定案】

每章（2500–4000 字）必須至少新增以下**至少一項**：

- 新規則（或舊規則的新層次）
- 新怪物／異常現象（或已知怪物的新能力）
- 新角色（或已知角色的新面向）
- 新地點（或已知地點的新區域）
- 新伏筆（或舊伏筆的新線索）
- 新代價（或已知代價的升級）
- 新情感資訊（周倉與許小小的關係推進、周倉的心理變化）

如果一章寫完，上述七項無一新增——這一章要重做。

## 審稿清單

【已定案】

每章完成後，逐項檢查：

### 節奏
- [ ] 本章有沒有水字數的段落？（連續三句可刪不影響劇情 = 水）
- [ ] 主角有沒有主動行動？（不能整章都在反應）
- [ ] 衝突是否在升級？（不能原地打轉）

### 恐怖感
- [ ] 恐怖來源是不是規則／存在／變異（不是跳嚇或血腥）？
- [ ] 有沒有一個夠好的身體細節？（不是抽象形容詞）

### 對話
- [ ] 每個角色的對話聽起來像那個人嗎？
- [ ] 對話有沒有變成設定說明會？

### AI 腔
- [ ] 有沒有「他意識到」「他終於明白」？
- [ ] 有沒有段末總結句？
- [ ] 有沒有過度完整的修辭結構？

### 章末
- [ ] 結尾有沒有五種有效鉤子之一？
- [ ] 絕對不是「他走進了下一個房間」？

### 資訊增量
- [ ] 本章新增了什麼？（七類至少一項）
