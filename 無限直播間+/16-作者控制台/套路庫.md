# 套路庫

## 文件狀態

【已定案】

本文是《無限直播間+》的敘事套路庫。整理規則怪談、直播間、系統流、存在恐怖等類型的常見敘事模式，供寫作與審稿時參考。

本文不是設定文件——設定查閱 [[已定案設定總表]]。本文是「怎麼寫」的劇本工具箱。

## 一、規則怪談套路

### 核心閱讀快感

讀者讀規則怪談要的是一條認知曲線：

> 看見規則 → 感到不對勁 → 試探規則 → 付出代價 → 理解真正的規則 → 用新規則反制

### T1：假規則 → 真規則

最核心的規則怪談套路。系統／副本給出的表面規則不是真正的規則。

**節奏模板**：

```
階段 1：系統給出「規則」。
        ——看起來合理、工整、像官方說明。

階段 2：周倉注意到矛盾。
        ——規則與環境細節不符。
        ——規則本身有邏輯漏洞。
        ——規則的措辭有微妙的不協調（時態、人稱、缺字）。

階段 3：試探。
        ——小動作測試規則的邊界。
        ——可能付出小代價。

階段 4：真規則浮現。
        ——不是完整揭示，是「原來這一條是錯的」。
        ——新理解比舊規則更窄、更不舒適。

階段 5：代價。
        ——知道真規則不等於安全。
        ——真規則往往比假規則更殘酷。
```

**寫作要點**：
- 假規則要寫得像真的——太明顯的假規則沒有揭示快感。
- 真規則不要一次講完。一次只推翻一條。
- 最好的規則反轉：讀者以為自己已經知道真規則了，然後發現那也只是第二層假規則。

### T2：規則的漏洞不是 bug，是陷阱

系統規則有漏洞——但這個漏洞不是作者寫漏了，是系統故意留的。

**範例**：規則說「不得離開便利店」，但沒說不能把手伸出去。周倉伸手——發現伸手的部分開始被抹消。

**寫作要點**：
- 讀者要和周倉一起發現漏洞。
- 漏洞的代價要夠具體、夠恐怖。
- 有些漏洞是真的（可以用），有些是陷阱（用了就死）。區分兩者是周倉的核心技能。

### T3：規則之間互相矛盾

兩條規則同時成立，執行其中一條就會違反另一條。

**範例**：「主播必須完成副本指定的任務。」+「主播不得進入地下三層。」任務物品在地下三層。

**寫作要點**：
- 矛盾不是邏輯遊戲——是壓力裝置。周倉必須在兩條規則之間的縫隙找到第三條路。
- 矛盾規則的解決方式，應揭示世界觀的深層規則。

### T4：規則的措辭本身就是線索

- 時態變化：「不得回頭」→「不得回頭過」
- 人稱變化：「主播不得…」→「你不得…」
- 語體變化：規則突然從系統腔變成口語腔
- 缺字／亂碼：規則的中間或結尾被系統刪過

**寫作要點**：
- 措辭變化要夠微妙——第一眼可能看不出來。
- 不要讓周倉立刻解釋這些變化。讓他注意到，然後在數章後才理解含義。

## 二、直播間套路

### T5：彈幕的多層功能

彈幕不是背景噪音。彈幕有五種功能：

| 功能 | 說明 | 範例 |
|---|---|---|
| **提示** | 給周倉他不知道的資訊 | 觀眾看到周倉看不到的角度 |
| **誤導** | 系統偽彈幕或惡意觀眾的假資訊 | 慫恿周倉做危險的事 |
| **世界觀側寫** | 彈幕內容間接揭示世界規則 | 觀眾對某件事的反應比事件本身更有資訊量 |
| **情感連結** | 許小小的彈幕殘留、陳述碎片 | 一句話的重量超過整章的描述 |
| **壓力裝置** | 觀眾的期待本身就是壓力 | 數十萬人看著你做選擇 |

**寫作要點**：
- 同一條彈幕可以在不同時間點有不同的含義（伏筆）。
- 彈幕數量控制：關鍵時刻可以密集（彈幕刷屏），日常時刻零散。不要每段都插彈幕。
- 彈幕的語氣多樣性：有認真的、有搞笑的、有髒話的、有破碎的。全部工整 = 假。

### T6：觀眾不是被動的——他們有立場

不同觀眾看到不同的真相（[[碎片立場理論]]）。

**寫作要點**：
- 同一事件發生後，彈幕反應要分歧。有人罵周倉蠢，有人說他做得對，有人說第三種完全不同的解讀。
- 周倉必須從矛盾資訊中推理，而不是單挑一個「正確」的彈幕。
- 高權重觀眾的彈幕語氣要有微妙不同——更完整、更像在「引導」而非「反應」。

### T7：直播間的「觀眾期待」可作為壓力

周倉不只是在被系統考驗——他同時在被數十萬觀眾看著。他們在等他犯錯、等他精彩、等他死。

**寫作要點**：
- 在關鍵選擇前，放一兩條彈幕代表觀眾期待。
- 周倉的心理壓力不只是來自規則和怪物——來自「被看著做選擇」。
- 後期周倉反向利用這點：故意讓觀眾看到他想讓他們看到的。

## 三、系統流套路

### T8：系統不是中立的

系統有目的、有立場、甚至有偏好。它不是一個工具——它是一個角色。

**寫作要點**：
- 系統的態度透過規則措辭、UI 細節、偽彈幕、管理員行動間接體現。
- 不要在早期就揭示系統的完整意圖。讓周倉（和讀者）逐步拼出系統是什麼。
- 系統不「邪惡」——它只是用一種讀者直覺上覺得不對的方式運作。

### T9：UI 異常是線索

系統 UI 上的異常（閃爍、亂碼、錯位、殘影）不是 bug——是系統無法完全抹消的痕跡。

**寫作要點**（與 F-0217 邊緣權限聯動）：
- 周倉看到的 UI 殘影不能太多（每章一次是上限）。
- 每次殘影都要有意義——要麼揭示新資訊，要麼強化某個懸念。
- 殘影的內容不要在當下完全解釋。留到後續。

### T10：積分／貨幣／權限的遞進

- 積分不是數字遊戲。積分是周倉能做什麼的限制器。
- 商城情報可能被汙染——買到的資訊不一定是真的。
- 權限提升的同時，系統對周倉的關注也在提升。

## 四、存在恐怖套路

### T11：抹消 > 死亡

《無限直播間+》的恐怖核心是「存在校正」——不是死亡，是被刪除到從來沒有存在過。

**寫法規則**：
- 抹消的恐怖不在過程，在結果——一個人消失了，但更可怕的是「她從來沒有存在過」的暗示。
- 周倉的記憶損失（救許小小後）就是存在恐怖——他感受到「少了什麼」但不知道少了什麼。
- 不要讓角色討論「存在主義」。讓存在恐怖透過細節浮現：一張照片上少了一個人、一個名字寫不出來、想不起一個人的聲音。

### T12：存續人格的悲劇感

存續人格不是「活下來了」——是「被系統判定應該結束，但還在」。他們永遠在系統的校正威脅下。

**寫法規則**：
- 許小小的存在本身就有悲劇感——不是因為她受苦，是因為她「不應該存在」卻還在。
- 存續人格的角色不應該整天悲傷。他們的存在就是反抗。悲傷是讀者的感受，不是角色的台詞。

### T13：身體細節的恐怖

熟悉的身體出現不該有的變化。

**寫法規則**：
- 不要寫「他的手變得很恐怖」。寫「他的無名指根部沒有傷口——那根手指好像從來沒有長出來過。但他記得今天早上用它按過電梯。」
- 身體恐怖的節奏：一個細節 → 角色否認 → 確認 → 更大的細節 → 無法否認。
- 身體恐怖不是為了噁心讀者——是為了讓抹消變成一種可以被感官感知的具體事件。

## 五、反轉套路

### T14：盟友反轉

某個看似幫助周倉的人／彈幕／情報源，其實在引導他走向某個方向。

**寫法規則**：
- 反轉前要有足夠線索——讓讀者重讀時能發現。
- 反轉後不要讓反派「暴露真面目」開始解釋。保持模糊——讀者不確定是不是真的反轉。
- 最好的反轉：反轉後讀者回去看前面的互動，發現每一句話都有雙重含義。

### T15：規則反轉

讀者以為的規則被推翻。見 T1。

### T16：身份反轉

某人不是他聲稱的那個人。

**寫法規則**：
- 《無限直播間+》最有效的身份反轉：不是「A 其實是 B」，而是「A 不存在——他的身份是在系統中被拼湊出來的」。
- 高權重觀眾模仿許小小、林█、陳述（T17）就是此類反轉。
- 身份反轉後，不要讓角色立刻知道全部真相。留一點模糊。

### T17：模仿者反轉

高權重觀眾可以模仿關鍵角色的語氣。周倉收到的「許小小的彈幕」可能不是她。

**寫法規則**：
- 模仿彈幕要有極細微的破綻——一個不該用的詞、一個不對的語氣、一個錯位的記憶細節。
- 破綻不要立刻揭穿。讓周倉（和讀者）懷疑但不確定。

## 六、情感套路

### T18：未完成交易的重量

周倉和許小小之間的「未完成交易」是全書最重要的情感引擎。它不是一個設定——是每一段情感互動的底層邏輯。

**寫法規則**：
- 不要讓周倉時時刻刻想著許小小——那會讓情感變廉價。
- 讓情感透過物品（牛奶盒）、習慣（想幫人打開東西）、空白（記得要找一個人但想不起名字）浮現。
- 「未完成」的意思是：線還在，但人不在。寫法上要保持這種「有連結但觸不到」的質地。

### T19：記憶損失的情感使用

周倉救許小小後失去部分記憶。這是情感引擎，不是失憶梗。

**寫法規則**：
- 記憶損失不是「忘了某件事」。是「記得某個感覺但不知道來源」。
- 每次周倉用未登記能力尋找許小小，就失去一段關於她的記憶。這是核心矛盾——越想找，越忘記為什麼要找。
- 不要讓記憶恢復成為解決方案。記憶損失是代價，不是謎題。

## 七、章末鉤子庫

### 具體範例（對應行文風格規範的五種類型）

| 類型 | 弱（不要用） | 強（可以用） |
|---|---|---|
| **新規則暴露** | 「他發現了一條新規則。」 | 「規則四的後半句有一個字是反白的。不是 UI 設計——是有人在系統文字下面蓋了一層新的。」 |
| **代價揭露** | 「他為自己的選擇付出了代價。」 | 周倉低頭看手。無名指的位置是平的。不是傷口，不是疤痕——是指頭從來沒有存在過。他記得今天早上用這根手指按了電梯。 |
| **身份反轉** | 「他突然意識到這個人不是真的。」 | 「你不是第一個 F-0217 來這裡的人。」說話的人穿著管理員制服，但左胸口袋上的編號是倒過來的。 |
| **行動倒數** | 「時間不多了。」 | 終端刷新。定位完成度：97%。周倉的影子在牆上轉頭看他。 |
| **情感重擊** | 「他想起了許小小。」 | 彈幕飄過一行字。只有四個字。「打開了嗎。」不是問號——是句號。這不是許小小。這是記得許小小說過這句話的東西。 |

## 使用規則

【已定案】

1. 寫新章節前，查閱本文件確認本單元應用的套路類型。
2. 不可在同一章內堆疊超過 3 種套路。套路是工具，不是清單。
3. 每個套路都可以變體——不要機械套用。讀者能聞到公式味。
4. 審稿時檢查：本章的套路是否執行到位？是否有足夠的具體細節（不是模板化執行）？
